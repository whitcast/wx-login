1.jssdk文档说明

阅读对象

本文阅读对象：使用jssdk构建自定义登录注册的前端用户。

jssdk例子访问地址：https://web.jiezhansifang.com/v2/public/index.html

1.1 术语

1.companyId 公司id，公司负责人在注册公司时由系统生成的公司唯一标识。

2.appType 接入的应用类型（1：PC端应用；2：移动端应用；3：H5应用）

3.appKey 公司下业务线的唯一标识。

4.appSecret 公司下业务线的密钥。

5.appChannel 用于下发推广的渠道

1.2 使用场景

1.希望自定义 登录、注册、绑定手机、找回密码、都三方登录 页面样式时。

2.不希望开发以上功能逻辑时。

jssdk封装了所有用户中心的逻辑实现比如（验证码功能、字段校验等等）。用户只需要按照例子将接口排列组合就可以得到想要的功能页面。

1.3 代码示例

==（以下例子仅供参考，请不要使用例子中的appkey、appChannel等参数，以免出现无法使用的情况）==

            <input id="password" class="row" placeholder="密码" />

        function getUrlParamByName(name) {
            var reg = new RegExp("(^|\\?|&)"+ name +"=([^&]*)(\\s|&|$)", "i");
            if (reg.test(location.href)) return unescape(RegExp.$2.replace(/\+/g, " "));
            return "";
        };

        var customAppKey = getUrlParamByName("appKey");
        var customCompanyId = getUrlParamByName("companyId");

        acrossSDK.config({
            companyId:customCompanyId||"6b310eb79c5b4bf5bafed4632a493fb6",
            appKey: customAppKey?customAppKey:"b1deb7cadff646e0915d69bb1d886254",
            appSecret: "",
            appType: "1",
            appChannel: "",
            actionList: [
                "login"
            ]
        });

        acrossSDK.ready(function() {
            //登录
            document.getElementById("submit").onclick = function(event) {
                var request = {
                    passport: document.getElementById("username").value,
                    password: document.getElementById("password").value,
                    vcode: document.getElementById("vcode").value || ""
                };

                function callback(res) {
                    if(res.status == "error" && res.code && res.code == 10405) {
                        document.getElementById("vcodeShell").style.display = "";
                        document.getElementById("loginVcode").src = acrossSDK.getLoginVcode();
                    }

                    //验证码错误需要重新刷新
                    if(res.status == "error" && res.code && res.code == 10406) {
                        document.getElementById("loginVcode").src = acrossSDK.getLoginVcode();
                    }

                    if(res.status == "success") {

                        document.getElementById("loginUser").text = res.data.passport || ""
                    }
                    alert(res.status + "：" + res.info);
                }
                acrossSDK.loginSubmit(request, callback);
            }

            document.getElementById("loginVcode").onclick = function(event) {
                document.getElementById("loginVcode").src = acrossSDK.getLoginVcode();
            }
        });
    </script>

</html>

2.用户中心通用页面

2.1 阅读对象

本文阅读对象：使用预定好的样式的登录注册页面的前端。

例子访问地址：https://web.jiezhansifang.com/v2/public/common/index.html

2.2 术语

1.closable:true|false,表示是否需要显示关闭按钮。

2.type:表示引入什么页面，比如：type:"smsLogin"就表示引入的是短信登录页面。

3.companyId 公司id，公司负责人在注册公司时由系统生成的公司唯一标识。

4.appType 接入的应用类型（1：PC端应用；2：移动端应用；3：H5应用）。

5.appKey 公司下业务线的唯一标识。

6.appChannel 用于下发推广的渠道。

2.3 使用场景

1.希望尽快投入使用，不需要自定义样式的页面的情况。

2.通用页面封装了所有用户中心的逻辑实现比如（验证码功能、字段校验等等）甚至一套通用皮肤。用户只需要按照例子使用即可得到即刻可以使用的登录、注册、找回密码、实名认证等等页面。
